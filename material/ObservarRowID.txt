He probado el siguiente flujo:

Fragment Receta: guardar Receta, observar rowid
Fragment Ingredintes: lista
Fragment Pasos: lista

Si retrocedo entre fragmentos, edito los valores y 
navego se duplica la receta.

Por lo tanto, el flujo que implementaré será guardar al final todo.

Para observar en el viewmodel el rowid había hecho:

    public void setReceta(Receta receta) {
        this.receta.setValue(receta);

        //Observamos el resultado del rowId
        repo.insertarReceta(receta).observeForever(new Observer<Long>() {
            @Override
            public void onChanged(Long id) {
                receta.setIdReceta(id);
                CrearRecetaViewModel.this.receta.setValue(receta);
            }
        });

Y en repositorio:

    public LiveData<Long> insertarReceta(Receta receta) {
        MutableLiveData<Long> rowIdLiveData = new MutableLiveData<>();

        Recetario.servicioExecutor.execute(() -> {
            long id = mRecetaDAO.insertarReceta(receta);
            rowIdLiveData.postValue(id); // Notifica el resultado al hilo principal
        });

        return rowIdLiveData;
    }
